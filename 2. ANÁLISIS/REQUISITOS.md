## Requisitos identificados

### RQ-01: Registrar ventas
El sistema debe permitir registrar cada venta realizada en el restaurante, ya sea en mesa, para llevar o por delivery. Cada registro debe incluir los productos vendidos, cantidades, precio total y forma de pago.

### RQ-02: Mostrar inventario disponible
El sistema debe mostrar en todo momento el inventario actual de insumos y productos (ejemplo: carne, verduras, bebidas, pan). La información debe ser clara y fácil de consultar desde cualquier dispositivo autorizado.

### RQ-03: Alertar productos con baja existencia
El sistema debe enviar una alerta automática cuando un producto o insumo esté por acabarse. La alerta debe llegar al dueño o encargado antes de que el producto se termine por completo.

### RQ-04: Registrar pedidos pendientes
El sistema debe permitir llevar un registro de los pedidos que aún no han sido entregados al cliente, diferenciando entre pedidos en preparación, listos para servir y entregados.

### RQ-05: Consultar productos por nombre
El sistema debe permitir buscar cualquier producto o insumo escribiendo su nombre o parte de él. La búsqueda debe ser rápida y mostrar resultados coincidentes en menos de 3 segundos.

### RQ-06: Generar resumen básico de ventas
El sistema debe generar un resumen diario de ventas que incluya: cantidad de pedidos, ingreso total, productos más vendidos y método de pago más usado.

### RQ-07: Enviar pedidos a la cocina automáticamente
Cuando un mesero toma un pedido desde su tableta o celular, el sistema debe enviarlo automáticamente a una pantalla visible en la cocina, sin necesidad de que el mesero vaya físicamente a dejar el papel.

### RQ-08: Controlar tiempos de preparación
El sistema debe medir y mostrar cuánto tiempo lleva cada pedido desde que se envía a cocina hasta que se marca como listo. Si un pedido supera los 20 minutos sin estar listo, debe mostrar una alerta.

### RQ-09: Separar menú por categorías
El sistema debe organizar los productos del menú en categorías (entradas, platos fuertes, postres, bebidas) para que los meseros y clientes encuentren fácilmente lo que buscan.

### RQ-10: Registrar múltiples formas de pago en una misma venta
El sistema debe permitir que un cliente pague una misma cuenta combinando diferentes formas de pago (ejemplo: mitad en efectivo, mitad con tarjeta) y debe registrar correctamente cada parte.
