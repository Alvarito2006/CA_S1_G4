Descripción del proyecto​

El trabajo se enfocó en levantar requerimientos de negocios reales, analizar los problemas observados y construir una propuesta tecnológica defendible mediante una dinámica de negociación técnica.

Durante el proceso se entrevistaron dos negocios:

  • Restaurante Don Juan
  • Clínica dental

Después del análisis, el equipo eligió trabajar principalmente con Restaurante Don Juan, porque permitió identificar con más claridad problemas operativos relacionados con:

  • control de ventas
  • recepción y atención al cliente
  • organización de pedidos
  • eficiencia en procesos internos

Objetivo general​

Analizar las necesidades operativas de un negocio real para proponer una solución tecnológica viable y defenderla mediante un proceso de negociación técnica, aplicando comunicación asertiva, trabajo colaborativo y documentación estructurada.

Objetivos específicos​

  • Realizar entrevistas a negocios reales para identificar necesidades, problemas y oportunidades de mejora.
  • Analizar la información recopilada para convertirla en requisitos claros y priorizados.
  • Elaborar propuestas tecnológicas orientadas al negocio seleccionado.
  • Desarrollar un elevator pitch individual por integrante.
  • Simular una junta directiva técnica con discusión, negociación y toma de decisiones.
  • Publicar el proceso y los entregables en un sitio de documentación web.

Problema identificado​

En el restaurante se detectaron procesos manuales o poco estructurados que pueden provocar:

  • retrasos en la atención al cliente
  • desorganización en la toma y despacho de pedidos
  • dificultad para llevar control de ventas
  • poca visibilidad del estado operativo del negocio
  • mayor probabilidad de errores humanos

Propuesta general​

Como respuesta, el equipo planteó una solución digital orientada a apoyar la operación del restaurante mediante:

  • registro de ventas
  • organización de pedidos
  • control básico de atención al cliente
  • seguimiento de procesos internos
  • mejor orden en la recepción y despacho de solicitudes

Fases documentadas​

  1. Recolección de información
    Entrevistas, preguntas guía, justificación del caso principal y evidencia de trabajo de campo.

  2. Análisis
    Definición de requisitos y descripción del flujo actual del negocio.

  3. Propuesta
    Elevator pitches, junta directiva, acuerdos, flujo objetivo y postmortem.

Herramientas utilizadas​

  • Trello para planificación y seguimiento del trabajo
  • GitHub para control del repositorio
  • Markdown para redactar los entregables
  • YouTube y Google Drive para videos
  • Docusaurus para la publicación final de la documentación

Navegación recomendada​

  • Comienza en Preguntas de entrevista para ver cómo se levantó la información.
  • Continúa con Requisitos identificados para entender la traducción del problema a necesidades del sistema.
  • Cierra con Junta directiva y Postmortem para revisar la discusión y el aprendizaje final.
